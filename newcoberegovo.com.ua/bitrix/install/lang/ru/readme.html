Руководство по установке продукта "1С-Битрикс: .NET Forge"

  1. Подготовьте систему к установке продукта "1С-Битрикс: .NET Forge". Если вы устанавливаете продукт не с помощью полного инсталлятора, то удостоверьтесь, что ПО соответствует минимальным техническим требованиям продукта:

    • OC Windows XP Professional / Vista (кроме Home Basic) / 7 / Server 2003 / Server 2008;
    • установите и настройте службу IIS версии 5.1 и выше;
    • установите Microsoft ASP.NET Framework 3.5 и выше;


      • Версия Microsoft ASP.NET Framework, установленная на локальном компьютере, должна соответствовать версии, используемой на предполагаемом хостинге. В противном случае потребуется дополнительная настройка "1С-Битрикс: .NET Forge".

    • установите и настройте MS SQL Server 2005 Express, SQL Server 2005, SQL Server 2008.

  2. Создайте каталог. В этот каталог разархивируйте файлы из дистрибутива продукта "1С-Битрикс: .NET Forge".


  3. Добавьте новый веб-узел в IIS, указав путь до папки с распакованным дистрибутивом.


  4. Настройте права доступа для группы пользователей NETWORK SERVICE к папке, в которую распакован архив дистрибутива продукта.


  5. Откройте страницу http://localhost/<ваш_псевдоним>/install.aspx в браузере.


  6. Следуйте инструкциям Мастера установки.


    7. Первый шаг установки (начало установки):

    На первом шаге установки продукта выводится информации о продукте и его установке.

    Второй шаг установки (лицензионное соглашение):

    Выводится текст лицензионного соглашения. Вы должны ознакомиться с лицензионным соглашением и принять его условия.

    Третий шаг установки (лицензионный ключ):

    Eсли вы купили систему, введите полученный в письме лицензионный ключ. Если вы ставите продукт для ознакомления, то оставьте значение равным DEMO.

    Четвертый шаг установки (предварительная проверка):

    Проверка системы на соответствие минимальным техническим требованиям продукта, права доступа к диску, а также рекомендации по настройке системы для оптимальной работы.
    Если ваша система не соответствует минимальным техническим требованиям, то вверху экрана появится сообщение об ошибке (красным цветом), описывающая несоответствие. Более подробное описание несоответствия находится в теле страницы. Вы не сможете продолжить установку продукта до устранения несовместимостей.
    Если ваша система не соответствует рекомендованным настройкам системы, то дальнейшая инсталляция возможна. Список таких несоответствий отобразится в теле страницы. Однако после установки рекомендуется привести систему в соответствие с рекомендованными настройками.

    Пятый шаг установки (создание базы данных):

    Создание конфигурационного файла соединения с базой данных, загрузка данных в базу данных.

      Параметры базы данных:

      • Сервер: имя сервера, на котором установлен MS SQL (например, localhost), и имя экземпляра MS SQL (например, BITRIX), которое указывается при установке MS SQL.
      • Пользователь базы данных: переключатель определяет, создавать нового пользователя базы данных в процессе установки или же использовать данные существующего пользователя.
      • Использовать Windows-аутентификацию: опция позволяет подключаться к базе данных от имени учетной записи Windows, которую использует IIS. Обычно это NETWORK SERVICE.
      • Имя пользователя: имя (логин) пользователя MS SQL для доступа к базе данных.
      • Пароль: пароль пользователя для доступа к базе данных.
      • База данных: переключатель определяет создавать новую базу данных в процессе установки или же использовать существующую.
      • Имя базы данных (Новая база данных): имя базы данных (только латинскими символами), в которую будет установлен продукт.

      Параметры администратора базы данных:

      • Использовать Windows-аутентификацию: опция позволяет подключаться к базе данных от имени учетной записи Windows, которую использует IIS. Обычно это NETWORK SERVICE.
      • Логин: логин администратора для подключения к базе данных.
      • Пароль: пароль администратора для подключения к базе данных.

      Дополнительные параметры:

      • Очистить БД перед установкой: опция позволяет очистить существующую базу данных перед установкой.


    Шестой шаг установки (установка продукта):

    Автоматический шаг, когда система создает таблицы в базе данных и устанавливает файлы системы. После завершения процесса создания базы данных система автоматически перейдет к следующему шагу.

    Седьмой шаг установки (создание администратора):

    Создание бюджет администратора сайта. Под этим бюджетом будут доступны все функции настройки и управления сайтом. После установки системы вы сможете создать дополнительных пользователей с меньшим количеством прав.

    • Отображаемое имя: псевдоним администратора сайта.
    • Логин: логин (имя) администратора сайта для входа в административный раздел. Логин должен быть не короче трех символов и содержать только латинские буквы и цифры.
    • Пароль: пароль администратора сайта должен быть не короче семи символов. Используйте в пароле только латинские символы и цифры, причем один из символов должен быть не алфавитно-цифровым (например: *, ~).
    • Подтверждение пароля: введите пароль еще раз для проверки правильности набора.
    • E-Mail: адрес электронной почты администратора сайта.
    • Имя: имя администратора сайта.
    • Фамилия: фамилия администратора сайта.


    Восьмой шаг (настройка почты):

    Настройка параметров и проверка отправки почты.

    • Адрес SMTP-сервера: адрес почтового SMTP-сервера.
    • Порт SMTP-сервера: порт сервера. Порт зависит от типа соединения, 25 - для обычного и 465 - для зашифрованного (с использованием SSL). Если указать порт 465, то необходимо отметить опцию Использовать SSL.
    • Логин: логин владельца почтового ящика.
    • Пароль: пароль владельца почтового ящика.
    • Email отправителя по умолчанию: адрес электронной почты администратора портала.
    • Email получателя тестового письма: адрес электронной почты получателя тестового письма.

    Для проверки корректности введеных параметров надмите кнопку Отправить тестовый email. Если на указанный в поле Email получателя тестового письма пришло тестовое письмо, то настройка почты выполнена корректно.

    Окончание работы мастера:

    После завершения установки система перейдет к Мастеру настройки сайта.

  7. Следуйте инструкциям Мастера настройки.

    Первый шаг мастера:

    На первом шаге выбирается одно из доступных решений для вашего сайта. Решения отличаются внешним оформлением, представленной информацией на главной странице и базовыми настройками.

    Второй шаг мастера:

    На втором шаге работы мастера выбирается цветовая тема шаблона для выбранного на первом шаге решения. Для разных решений предлагаются разные цветовые темы.

    Третий шаг мастера:

    На данном этапе задаются настройки для выбранного вами решения и происходит установка заданных настроек.

    Обратите внимание: количество последующих этапов на данном шаге зависит от выбранного решения и для каждого решения будет различается.

    Окончание работы мастера:

    После завершения настройки выводится информация об успешном завершении работы Мастера настройки сайта.
